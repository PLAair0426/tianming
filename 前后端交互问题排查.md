# 前后端交互问题排查和修复

## 发现的问题

### 1. CORS配置冲突 ⚠️ **已修复**
**问题**: FastAPI不允许同时使用 `allow_credentials=True` 和 `allow_origins=["*"]`

**修复**: 
- 将 `allow_credentials` 改为 `False`
- 保持 `allow_origins=["*"]` 以允许所有来源

### 2. 前端错误处理改进 ✅ **已修复**
**改进**:
- 添加了详细的调试日志
- 改进了错误信息显示
- 添加了请求/响应的日志输出

## 修复后的配置

### 后端 CORS 配置 (`backend/api_main.py`)
```python
if DEBUG_MODE:
    app.add_middleware(
        CORSMiddleware,
        allow_origins=["*"],  # 允许所有来源
        allow_credentials=False,  # 必须为False才能使用["*"]
        allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS", "PATCH"],
        allow_headers=["*"],
        expose_headers=["*"],
    )
```

### 前端请求配置 (`frontend/services/apiService.ts`)
```typescript
const response = await fetch(url, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  mode: 'cors',
  credentials: 'omit',  // 不使用credentials
});
```

## 重要：需要重启后端服务

**修改CORS配置后，必须重启后端服务才能生效！**

### 重启步骤：
1. 找到运行后端服务的终端窗口
2. 按 `Ctrl + C` 停止服务
3. 重新启动：
   ```bash
   cd backend
   python start_server.py
   ```
4. 等待看到 "Uvicorn running on http://0.0.0.0:8000"
5. **保持终端窗口打开**
6. 刷新前端页面

## 如果仍然无法连接

### 检查清单：
1. ✅ **后端服务是否运行？**
   - 访问 http://localhost:8000/docs
   - 检查终端窗口是否显示 "Uvicorn running"

2. ✅ **后端服务是否已重启？**
   - 修改CORS配置后必须重启
   - 检查终端窗口是否显示最新的日志

3. ✅ **前端开发服务器端口是什么？**
   - 检查浏览器地址栏的端口号（通常是3000或5173）
   - 确认后端CORS配置包含该端口

4. ✅ **浏览器控制台显示什么？**
   - 打开F12开发者工具
   - 查看Console标签的错误信息
   - 查看Network标签的请求状态

5. ✅ **防火墙是否阻止连接？**
   - Windows防火墙可能阻止localhost连接
   - 尝试临时关闭防火墙测试

## 常见错误

### 错误1: "Failed to fetch"
**原因**: 后端服务未运行或网络问题
**解决**: 启动后端服务并检查网络连接

### 错误2: "CORS policy" 错误
**原因**: CORS配置不正确
**解决**: 
1. 确认后端已重启
2. 检查 `allow_credentials=False`
3. 检查 `allow_origins=["*"]`

### 错误3: "404 Not Found"
**原因**: API路径错误
**解决**: 检查API路径是否为 `/api/divination/generate-line`

### 错误4: "500 Internal Server Error"
**原因**: 后端服务器内部错误
**解决**: 查看后端终端窗口的错误日志
