# 用户隔离功能测试指南

## 🎯 测试目标

验证用户隔离功能是否正常工作：
- ✅ 每个用户有独立的元气系统
- ✅ 用户之间互不影响
- ✅ 用户状态会保持

---

## 📋 测试前准备

### 1. 确认部署完成

1. 访问 [Render Dashboard](https://dashboard.render.com/)
2. 进入服务：`tianming`
3. 查看 **Logs** 标签
4. 确认看到 "Deploy successful" 或 "Your service is live"
5. 等待 2-3 分钟让服务完全启动

### 2. 确认服务正常运行

在浏览器访问：
```
https://tianming.onrender.com/health
```

应该返回：
```json
{"status": "ok"}
```

---

## 🧪 测试步骤

### 测试 1：验证用户隔离（核心测试）

#### 步骤 1：用户A 测试

1. **打开前端网站**
   - 访问：`https://www.godbar.dpdns.org`
   - 或：`https://tianming-b5z.pages.dev`

2. **查看初始元气状态**
   - 打开浏览器开发者工具（F12）
   - 切换到 **Console** 标签
   - 查看元气状态，应该是 **100%**

3. **进行一次占卜**
   - 输入问题并完成占卜
   - 观察元气消耗
   - 记录消耗后的元气值（例如：80%）

4. **刷新页面**
   - 刷新后，元气值应该保持（不是重置为100%）
   - 这证明状态已保存

#### 步骤 2：用户B 测试（不同设备/网络）

1. **使用不同设备或网络**
   - 使用手机（移动网络）
   - 或使用另一个浏览器（无痕模式）
   - 或使用 VPN 切换 IP

2. **访问前端网站**
   - 打开前端网站
   - 查看元气状态
   - **应该显示 100%**（不受用户A影响）

3. **进行占卜**
   - 完成一次占卜
   - 观察元气消耗
   - 用户B的消耗不应该影响用户A

#### 步骤 3：验证隔离

1. **回到用户A的设备**
   - 刷新页面
   - 查看元气状态
   - **应该仍然是之前的值**（例如：80%）
   - 不受用户B的影响

2. **结论**
   - ✅ 如果用户A和用户B的元气值独立 → 隔离成功
   - ❌ 如果用户B的消耗影响了用户A → 隔离失败

---

### 测试 2：API 直接测试

#### 使用 curl 或 Postman 测试

**测试用户A（IP: 模拟）**：
```bash
# 获取元气状态
curl https://tianming.onrender.com/api/divination/karma-status

# 应该返回初始状态（100%）
```

**测试用户B（不同IP）**：
```bash
# 使用不同的 User-Agent 或 IP
curl -H "X-Forwarded-For: 2.2.2.2" \
     https://tianming.onrender.com/api/divination/karma-status

# 应该也返回初始状态（100%），不受用户A影响
```

---

### 测试 3：并发测试

#### 模拟多用户同时使用

1. **打开多个浏览器标签**
   - 每个标签代表一个用户
   - 使用无痕模式创建独立会话

2. **同时进行占卜**
   - 每个标签都进行占卜
   - 观察是否互相影响

3. **验证结果**
   - 每个标签的元气值应该独立变化
   - 不应该出现"元气不足"的错误（除非该用户自己的元气用完）

---

## 🔍 验证检查点

### ✅ 成功标志

1. **独立状态**
   - 不同用户看到不同的元气值
   - 用户A消耗不影响用户B

2. **状态保持**
   - 刷新页面后，元气值保持不变
   - 不会重置为100%

3. **正常功能**
   - 占卜功能正常工作
   - 元气消耗正常计算
   - 没有错误信息

### ❌ 失败标志

1. **共享状态**
   - 所有用户看到相同的元气值
   - 用户A消耗后，用户B也受影响

2. **状态丢失**
   - 刷新页面后，元气值重置为100%
   - 状态不保持

3. **功能异常**
   - 出现错误信息
   - 占卜功能无法使用

## 🐛 问题排查

### 如果测试失败

#### 1. 检查 Render Logs

1. 访问 Render Dashboard → Logs
2. 查看是否有错误信息
3. 查找关键词：
   - "error"
   - "exception"
   - "traceback"

#### 2. 检查代码是否正确部署

1. 确认代码已推送到 GitHub
2. 确认 Render 已重新部署
3. 查看部署日志确认部署成功

#### 3. 检查 IP 获取

在浏览器控制台查看：
```javascript
// 查看请求头
fetch('https://tianming.onrender.com/api/divination/karma-status')
  .then(r => r.json())
  .then(console.log)
```

#### 4. 清除缓存

- 清除浏览器缓存
- 使用无痕模式测试
- 重启浏览器
