# 防止单用户占用资源方案

## 🔍 问题分析

### 当前已解决的问题

✅ **元气共享问题**（已解决）
- 通过用户隔离，每个用户有独立的元气系统
- 用户A消耗完自己的元气，不影响用户B
- 用户B仍然有100%元气可以使用

### 仍可能存在的问题

⚠️ **资源占用问题**（需要解决）
- 用户A频繁请求，可能占用大量服务器资源
- 可能导致：
  - CPU/内存耗尽
  - DeepSeek API 调用过多（触发限流）
  - 服务响应变慢
  - 其他用户无法正常使用

## ✅ 解决方案：API 限流

### 方案 1：使用 slowapi（推荐，简单）

**slowapi** 是 FastAPI 的限流扩展，简单易用。

#### 安装依赖

在 `backend/requirements.txt` 添加：
```
slowapi>=0.1.9
```

#### 实现代码

在 `backend/api_main.py` 中添加限流：

```python
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded

# 创建限流器
limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

# 在路由中使用
@router.post("/divination/interpret")
@limiter.limit("10/minute")  # 每分钟最多10次
async def interpret_divination(request: DivinationRequest, req: Request):
    # ...
```

### 方案 3：自定义限流中间件（轻量级，无需额外依赖）

如果不想添加依赖，可以自己实现简单的限流。

#### 实现代码

在 `backend/app/routers/divination.py` 中添加：

```python
from collections import defaultdict
from time import time

# 存储每个IP的请求记录
request_records = defaultdict(list)

def check_rate_limit(client_ip: str, max_requests: int = 10, window_seconds: int = 60) -> bool:
    """
    检查是否超过限流
    :param client_ip: 客户端IP
    :param max_requests: 时间窗口内最大请求数
    :param window_seconds: 时间窗口（秒）
    :return: True表示允许，False表示超过限流
    """
    current_time = time()
    
    # 清理过期记录（超过时间窗口的请求）
    request_records[client_ip] = [
        req_time for req_time in request_records[client_ip]
        if current_time - req_time < window_seconds
    ]
    
    # 检查是否超过限制
    if len(request_records[client_ip]) >= max_requests:
        return False
    
    # 记录本次请求
    request_records[client_ip].append(current_time)
    return True

# 在路由中使用
@router.post("/divination/interpret")
async def interpret_divination(request: DivinationRequest, req: Request):

            status_code=429,

                "error": "请求过于频繁",
                "message": "请稍后再试，每分钟最多10次请求",
                "retry_after": 60
```

## 🔧 完整实现代码

### 在 divination.py 中添加限流

```python
from collections import defaultdict
from time import time

# 存储每个IP的请求记录（用于限流）
request_records = defaultdict(list)

def check_rate_limit(
    client_ip: str, 
    max_requests: int = 10, 
    window_seconds: int = 60
) -> tuple[bool, dict]:
    """
    检查是否超过限流
    :param client_ip: 客户端IP
    :param max_requests: 时间窗口内最大请求数
    :param window_seconds: 时间窗口（秒）
    :return: (是否允许, 限流信息)
    """
    current_time = time()
    
    # 清理过期记录
    request_records[client_ip] = [
        req_time for req_time in request_records[client_ip]
        if current_time - req_time < window_seconds
    ]
    
    # 检查是否超过限制
    if len(request_records[client_ip]) >= max_requests:
        # 计算需要等待的时间
        oldest_request = min(request_records[client_ip])
        retry_after = int(window_seconds - (current_time - oldest_request))
        
        return False, {
            "max_requests": max_requests,
            "window_seconds": window_seconds,
            "current_requests": len(request_records[client_ip]),
            "retry_after": retry_after
        }
    
    # 记录本次请求
    request_records[client_ip].append(current_time)
    return True, {}

# 在路由中使用
@router.post("/divination/interpret")
async def interpret_divination(request: DivinationRequest, req: Request):
    client_ip = get_client_ip(req)
    
    # 检查限流（每分钟最多10次）
    allowed, limit_info = check_rate_limit(client_ip, max_requests=10, window_seconds=60)
    if not allowed:
        raise HTTPException(
            status_code=429,
            detail={
                "error": "请求过于频繁",
                "message": f"请稍后再试，每分钟最多10次请求。请在 {limit_info['retry_after']} 秒后重试。",
                "limit_info": limit_info
            }
        )

```

## 🛡️ 多层保护策略

### 1. API 限流（防止频繁请求）

限制每个IP的请求频率，防止单个用户占用过多资源。

### 2. 元气系统（防止滥用）

已有的元气系统可以防止用户频繁使用：
- 频繁使用消耗更多元气
- 元气耗尽后无法使用
- 需要等待恢复

### 3. 请求超时（防止长时间占用）

为AI调用添加超时，防止单个请求占用过长时间。

### 4. 并发限制（可选）

限制同时处理的请求数量，防止资源耗尽。

1. **高优先级**：API 限流
