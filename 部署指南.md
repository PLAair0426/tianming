# Cloudflare 部署指南

## 📋 部署方案概述

由于 Cloudflare Workers 不支持 Python，建议采用**混合部署**方案：

- **前端**：部署到 Cloudflare Pages（免费、快速、全球 CDN）
- **后端**：部署到支持 Python 的平台（Railway、Render、Fly.io 等）

---

## 🎨 前端部署到 Cloudflare Pages

### 方法一：通过 Cloudflare Dashboard（推荐）

1. **准备构建**
   ```bash
   cd frontend
   npm install
   npm run build
   ```
   构建产物会在 `frontend/dist` 目录

2. **在 Cloudflare Dashboard 部署**
   - 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
   - 进入 **Pages** → **Create a project**
   - 选择 **Upload assets**
   - 上传 `frontend/dist` 目录中的所有文件
   - 项目名称：`celestial-divination`（或自定义）

3. **配置环境变量**
   - 在项目设置中，找到 **Environment variables**
   - 添加变量：
     ```
     VITE_API_BASE_URL = https://your-backend-url.com
     ```
   - ⚠️ **注意**：Vite 的环境变量必须以 `VITE_` 开头，且需要在构建时注入
   - 如果使用环境变量，需要重新构建并上传

### 方法二：通过 Git 连接（自动部署）

1. **连接 Git 仓库**
   - 在 Cloudflare Pages 中选择 **Connect to Git**
   - 授权访问你的 GitHub/GitLab 仓库
   - 选择仓库和分支

2. **构建配置**
   - **Build command**: `cd frontend && npm install && npm run build`
   - **Build output directory**: `frontend/dist`
   - **Root directory**: `/`（项目根目录）

3. **环境变量配置**
   - 在 **Environment variables** 中添加：
     ```
     VITE_API_BASE_URL = https://your-backend-url.com
     ```
   - 每次构建时会自动注入

### 方法三：使用 Wrangler CLI

```bash
# 安装 Wrangler
npm install -g wrangler

# 登录
wrangler login

# 在 frontend 目录下构建
cd frontend
npm run build

# 部署到 Pages
wrangler pages deploy dist --project-name=celestial-divination
```

## 🐍 后端部署选项

### 选项 1：Railway（推荐，简单易用）

1. **注册并登录** [Railway](https://railway.app/)
2. **创建新项目** → **Deploy from GitHub repo**
3. **选择仓库**，Railway 会自动检测 Python 项目
4. **配置环境变量**：
   ```
   DEEPSEEK_API_KEY=your_api_key_here
   DEBUG_MODE=False
   ```
5. **设置启动命令**：
   ```
   cd backend && uvicorn api_main:app --host 0.0.0.0 --port $PORT
   ```
6. **获取部署 URL**（例如：`https://your-app.railway.app`）

### 选项 2：Render

1. **注册并登录** [Render](https://render.com/)
2. **创建 Web Service** → **Connect GitHub**
3. **配置**：
   - **Build Command**: `cd backend && pip install -r requirements.txt`
   - **Start Command**: `cd backend && uvicorn api_main:app --host 0.0.0.0 --port $PORT`
   - **Environment**: `Python 3`
4. **添加环境变量**：
   ```
   DEEPSEEK_API_KEY=your_api_key_here
   DEBUG_MODE=False
   ```
5. **获取部署 URL**

### 选项 3：Fly.io

1. **安装 Fly CLI**：
   ```bash
   powershell -Command "iwr https://fly.io/install.ps1 -useb | iex"
   ```
2. **登录**：
   ```bash
   fly auth login
   ```
3. **在 backend 目录创建 fly.toml**（见下方配置）
4. **部署**：
   ```bash
   cd backend
   fly deploy
   ```

### 选项 4：Vercel（使用 Serverless Functions）

需要将 FastAPI 转换为 Vercel 兼容格式，或使用 `vercel-python`。

## 🔧 后端部署配置文件

### Railway 配置（railway.json）

在 `backend` 目录创建 `railway.json`：

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "uvicorn api_main:app --host 0.0.0.0 --port $PORT",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

### Fly.io 配置（fly.toml）

在 `backend` 目录创建 `fly.toml`：

```toml
app = "celestial-divination-api"
primary_region = "hkg"

[build]

[env]
  PORT = "8080"
  DEBUG_MODE = "False"

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = true
  auto_start_machines = true
  min_machines_running = 0
  processes = ["app"]

[[services]]
  protocol = "tcp"
  internal_port = 8080

  [[services.ports]]
    port = 80
    handlers = ["http"]
    force_https = true

  [[services.ports]]
    port = 443
    handlers = ["tls", "http"]
```

### Render 配置（render.yaml）

在项目根目录创建 `render.yaml`：

```yaml
services:
  - type: web
    name: celestial-divination-api
    env: python
    buildCommand: cd backend && pip install -r requirements.txt
    startCommand: cd backend && uvicorn api_main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: DEEPSEEK_API_KEY
        sync: false
      - key: DEBUG_MODE
        value: False
```

## 🔗 连接前后端

### 1. 更新前端 API 地址

部署后端后，获取后端 URL（例如：`https://your-api.railway.app`）

### 2. 配置前端环境变量

**如果使用 Cloudflare Pages Dashboard 上传**：
- 需要修改 `frontend/services/apiService.ts`，硬编码生产环境 URL
- 或在构建时设置环境变量

**如果使用 Git 连接自动部署**：
- 在 Cloudflare Pages 项目设置中添加环境变量：
  ```
  VITE_API_BASE_URL = https://your-api.railway.app
  ```

### 3. 更新后端 CORS 配置

在 `backend/api_main.py` 中，更新生产环境的 CORS 配置：

```python
allow_origins=[
    "https://your-frontend.pages.dev",  # Cloudflare Pages 域名
    "https://your-custom-domain.com",   # 自定义域名（如果有）
]
```

## 📝 注意事项

1. **环境变量**：Vite 的环境变量必须在构建时注入，运行时无法读取 `.env` 文件
2. **CORS**：确保后端允许前端域名的跨域请求
3. **HTTPS**：生产环境必须使用 HTTPS
4. **API 密钥**：不要在代码中硬编码 API 密钥，使用环境变量
5. **冷启动**：Serverless 平台可能有冷启动延迟，考虑使用常驻实例

## 🆘 常见问题

### Q: 前端无法连接后端？
A: 检查：
- 后端 URL 是否正确
- CORS 配置是否允许前端域名
- 后端服务是否正常运行

### Q: 环境变量不生效？
A: Vite 环境变量必须以 `VITE_` 开头，且需要在构建时设置

### Q: 构建失败？
A: 检查 Node.js 版本和依赖安装是否正常
